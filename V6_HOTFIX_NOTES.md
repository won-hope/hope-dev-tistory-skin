# 긴급 핫픽스 (V6 Hotfix)

라이브 티스토리 블로그 적용 시 발견된 치명적 회귀 버그(Regression)들을 해결했습니다.

## 1. 상단 메뉴(Category) 유실 문제 해결
- **이슈**: "분류 전체보기" 텍스트를 숨겼을 때, 티스토리 기본 DOM인 `.tt_category` 컨테이너의 레이아웃이 무너지며 카테고리 전체가 사라짐.
- **해결**: `.tt_category`와 그 자식 `li`에 명시적인 `display: flex`와 `list-style: none`을 부여하여 티스토리 기본 CSS의 간섭을 완전히 차단하고 정상 노출되도록 강제함.

## 2. 자바스크립트 스코프 오류 교정 (카테고리/통계 먹통)
- **이슈**: V5 업데이트 시, `DOMContentLoaded` 이벤트 리스너가 비동기 통신 콜백 함수(`applyWeatherTheme`) 내부에 작성되어 영구적으로 실행되지 않음.
- **해결**: 모든 DOM 조작 로직(인기 카테고리 스크래핑, 총 방문자수/포스팅수 카운팅, 블로그 개설일 연산)을 안전한 최상단 독립 스코프로 이동.
- 이제 인기 카테고리에 실제 카테고리 이름과 포스트 개수가 정상 맵핑됩니다.

## 3. 손그림 로딩바(Preloader) 및 사이드바 복구
- **이슈**: 이전 버전에서 무한 로딩 이슈로 삭제되었던 로딩바가 완전히 유실되어 별모양 애니메이션이 뜨지 않음.
- **해결**: `skin.html` 최하단에 프리로더와 사이드바 DOM을 다시 주입. 무한 로딩을 완벽히 방지하기 위해 로딩 직후(0.5초 뒤) 강제로 DOM에서 소멸(`remove()`)시키는 안전 장치(Failsafe) 로직을 추가함.

## 4. 글 상세 페이지(Post Detail) 레이아웃 붕괴 픽스
- **이슈**: `body#tt-body-page .magazine-grid`를 억지로 block 처리하며 글 본문의 폭(Width) 제한이 풀려 화면이 깨짐.
- **해결**: 포스트 본문 래퍼(`.post-detail`)에 `width: 100%`, `box-sizing: border-box`, `clear: both` 속성을 부여하여 주변 사이드바나 플로팅 요소의 영향을 받지 않는 독립된 블록으로 격리시킴.
